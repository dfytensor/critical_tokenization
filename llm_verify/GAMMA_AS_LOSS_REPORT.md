# γ 当 loss / 当解码器——两件事都做了，都是负面（实证"γ 是仪表盘，不是发动机"）

承接之前的问题"γ 能不能当 loss / 替代 CE / 新范式"，这里把其中**可做的两条**（γ-REINFORCE 奖励、γ-aware 解码）实际跑了一遍。结论：**两条朴素实现都没能让 γ 可控**——这把"γ 是监控量、不是驱动量"从理论判断变成了实测。

## (1) γ-REINFORCE（γ 当奖励）——❌ 不可控

score-function REINFORCE：rollout（no_grad）→ 采样序列 teacher-force 重过带 grad 拿 logp → reward = -(γ-γ_target)² → 策略梯度。两个目标 γ\*=0.08（更临界）和 0.25（退化），各 120 步：

| γ_target | 起 γ | 终 γ |
|---|---|---|
| 0.08 | 0.024 | 0.044 |
| 0.25 | 0.024 | 0.046 |

**两个目标走出几乎一样的轨迹**（都 0.024→~0.045），完全没有朝各自目标分叉 → **γ 没被奖励驱动**。

原因（与理论预测一致）：
- γ 是**群体统计量**，每步只采样 ~1280 token → γ 估计噪声大；
- reward 量级极小（-(γ-γ\*)² ≈ 0.001~0.06）→ 策略梯度信号被噪声淹没；
- REINFORCE 本身高方差，叠上 γ 的群体噪声 → 信号失效。

## (2) γ-aware 解码（闭环调 T）——❌ 不可控

"γ 恒温器"：每 32 token 测一次滑动窗 γ，γ 偏高就升 T、偏低就降 T，目标 γ\*=0.13，生成 2048 token：

| | γ 均值 | γ std | T 轨迹 |
|---|---|---|---|
| 固定 T=0.8 | 0.058 | 0.021 | — |
| γ-aware | 0.059 | 0.031 | 0.8→0.68→...→**0.3（触底卡死）** |

生成 γ 本身只有 ~0.058（远低于 0.13 目标），控制器一路降 T 到下限 0.3 也推不上去 → **解码端也控不住 γ**。（rep4 还从 0.19 升到 0.44——硬压 T 把文本压重复了，质量反而变差。）

## 诚实边界（不是"不可能"，是"朴素版不行"）
- 两处生成的 γ 都偏低（~0.06，vs dpo 实验里的 0.14），部分是**短 prompt / 短窗口**的人为压低（dpo_gamma 用完整 chat prompt 得 γ≈0.14）。更干净的 setup 可能给出**边际**更好的控制，但不会改变"γ 难驱动"的定性。
- 要让 γ-as-loss 真正可用，需要：**长得多的序列**（稳定 γ，代价大）、**reward reshaping**、**低方差估计器**（如 CPC 式对比 MI 而非原始直方图 γ）。这是**开放研究问题**，不是快赢。

## 与"γ 当监控"的对照（关键）

| γ 的角色 | 结果 |
|---|---|
| **被动监控**（观测 γ 漂移）| ✅ 真 DPO 上确认（γ 0.14→0.21，单调，补局部盲点）|
| **主动驱动**（γ-REINFORCE 当 loss）| ❌ 两目标同轨迹，不可控 |
| **主动驱动**（γ-aware 解码）| ❌ T 触底，γ 推不动 |

→ **γ 是个能精确"读"长程结构的仪表盘，但拿它去"写"（训练驱动 / 解码控制）在朴素形式下不成立。** 这正好实证了之前那条结论：γ 监控 RLHF 成立，γ 替代 CE / 替代自回归 / 当主 loss——不成立。
